# 目录摘要模板

使用以下模板为每个分析过的目录生成摘要。根据实际情况调整内容，省略不适用的部分。

---

```markdown
## 📁 `{目录路径}` — {一句话描述目录职责}

**所属模块 / Assembly**：`{Assembly Definition 名称，若有}`

### 概述

{2-4 句话描述此目录的核心职责、在项目架构中的角色}

### 核心脚本

| 文件名        | 类型                                                        | 职责         |
| ------------- | ----------------------------------------------------------- | ------------ |
| `{文件名}.cs` | MonoBehaviour / ScriptableObject / 纯逻辑类 / Editor / 接口 | {一句话描述} |
| ...           | ...                                                         | ...          |

### 关键接口

```csharp
// 列出目录中最重要的公共类/接口签名（省略方法体）
public class {ClassName} : {BaseClass}
{
    public {ReturnType} {MethodName}({Params});
    public event Action<{Args}> {EventName};
    [SerializeField] private {Type} {fieldName};
}
```

### 依赖关系

- **引用**：`{本目录引用的其他目录/模块}` — {引用原因}
- **被引用**：`{引用本目录的其他目录/模块}` — {被引用原因}

### 备注

- {值得注意的设计模式、技术债务、TODO 等}
```

---

## 使用说明

- **类型列**：标注脚本的 Unity 类型分类，帮助快速理解脚本在引擎中的角色
- **关键接口**：只列出对外暴露的、其他模块会调用的公共 API，不要列出私有实现
- **依赖关系**：双向记录，便于理解模块间的耦合情况
- **备注**：记录 AI 或开发者在后续工作中需要特别注意的事项
