# 项目概览模板

使用以下模板生成项目全局概览文档，置于最终输出的开头。

---

```markdown
# 📋 项目概览：{项目名称}

## 基本信息

| 属性       | 值                                      |
| ---------- | --------------------------------------- |
| Unity 版本 | {版本号}                                |
| 渲染管线   | URP / HDRP / Built-in                   |
| 目标平台   | {iOS / Android / PC / 多平台}           |
| 脚本后端   | Mono / IL2CPP                           |
| .NET 版本  | {.NET Standard 2.1 / .NET Framework 等} |

## 架构概述

{3-5 句话概述项目的整体架构风格，如：
- 使用了哪种架构模式（MVC、ECS、事件驱动等）
- 是否有自研框架
- 核心游戏循环的组织方式}
```

---

## 使用说明

- **基本信息**：从 ProjectSettings 和 Packages/manifest.json 提取
- **目录结构**：只展示 `Assets/` 下的一级和关键二级目录，用简短文字标注职责
- **Assembly Definition**：若项目未使用 asmdef，注明"未使用 Assembly Definition"，并说明所有代码编译在 Assembly-CSharp 中
- **入口与启动流程**：追踪从场景加载到游戏可交互的完整初始化链
- **第三方依赖**：区分 Unity Package Manager 包和手动导入的插件
